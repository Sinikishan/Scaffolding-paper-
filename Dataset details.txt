Dataset Description Document
Agent-Orchestrated Decision Support for Logical Framework Development
1. Dataset Overview

This study does not utilise a conventional benchmark dataset such as UCI, Kaggle, Hugging Face, or publicly available machine learning repositories. Instead, the research relies on practitioner-generated artefacts and evaluation observations collected during the pilot implementation of the proposed agent-orchestrated decision-support framework for Logical Framework development.

The dataset used in this research is therefore classified as a:

Structured qualitative and semi-structured evaluation dataset
Human-generated program design artefact collection
Expert-reviewed Logical Framework evaluation dataset
2. Nature of the Data

The collected data consists of:

Dataset Component	Description
Program Design Inputs	Problem statements, intervention goals, stakeholder mappings, and contextual descriptions entered by participants
Logical Framework Drafts	First-draft Logical Framework matrices generated manually and through the proposed system
Evaluation Metrics	Structured assessment scores related to completeness, coherence, alignment, and clarity
Practitioner Feedback	Qualitative observations and survey responses regarding usability and confidence
Expert Review Assessments	Independent evaluation comments and rubric-based scoring provided by domain experts
3. Data Collection Process

The dataset was generated during the pilot evaluation phase of the proposed framework involving:

18 program practitioners
Education-focused non-governmental organisations
Program managers and design practitioners
Human-in-the-loop evaluation activities

Participants were asked to:

Develop Logical Frameworks manually using traditional template-based methods.
Develop Logical Frameworks using the proposed agent-orchestrated decision-support framework.
Provide feedback regarding usability, clarity, confidence, and perceived effectiveness.

Independent expert reviewers subsequently assessed the generated artefacts using structured rubrics focused on:

Logical coherence
Completeness
Stakeholder alignment
Outcome-indicator consistency
Evaluability

6. Evaluation Dimensions

The dataset supports evaluation across the following dimensions:

Dimension	Description
Efficiency	Time to first coherent Logical Framework draft
Completeness	Coverage of required LFA components
Coherence	Internal consistency across outputs, outcomes, and indicators
Stakeholder Alignment	Mapping between stakeholders and practice changes
User Confidence	Practitioner perception and usability
7. Ethical and Privacy Considerations

The dataset does not contain:

Patient information
Clinical data
Financial records
Sensitive personal identifiers

All practitioner participation was voluntary, and collected responses were anonymised before analysis. The dataset was used strictly for academic and research evaluation purposes.

8. Data Availability Statement

No publicly available benchmark dataset was used in this study. The dataset generated during the evaluation phase consists primarily of practitioner-generated Logical Framework artefacts and expert assessment observations developed specifically for this research.

Supporting materials and sample artefacts may be made available by the corresponding author upon reasonable academic request.
